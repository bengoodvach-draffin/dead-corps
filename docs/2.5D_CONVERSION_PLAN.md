# Dead Corps - 2.5D Conversion Plan
**Date:** February 13, 2026  
**Status:** FUTURE CONSIDERATION - Not implementing yet  
**Purpose:** Complete plan for converting from 2D to 2.5D with height gameplay

---

## 🎯 DECISION SUMMARY

### **User Requirements:**
- ✅ Height must affect gameplay (not just visual)
- ✅ Zombies climbing onto buildings
- ✅ Snipers on rooftops with vision advantage
- ✅ Ravines/terrain elevation
- ✅ Maintain consistent 2D art style

### **Recommended Approach:**
**Billboard 3D with Discrete Height Levels (Option 2A)**

**Why this over other options:**
- Pure 2D fake height: Too much custom code, fragile, ~40 hours work
- Full 3D with slopes: Overkill, complex pathfinding, ~60% overhaul
- Billboard 3D: Best balance - ~30% overhaul, ~3 weeks, uses engine features

---

## 🎨 THE CORE CONCEPT: Visual ≠ Collision

**Key Insight:** You can have 2D sprite visuals with 3D gameplay features!

```
VISUAL LAYER (what players see):
- 2D sprites rendered as Sprite3D billboards
- Always face camera
- Maintains art style consistency
- Hand-drawn, pixel art, whatever you want

GAMEPLAY LAYER (invisible to player):
- 3D collision boxes (StaticBody3D, CharacterBody3D)
- 3D navigation mesh for pathfinding
- Real Y coordinates for height
- 3D raycasting for vision/LOS
```

**Example Building:**
```
BuildingNode (Node3D)
├── Sprite3D ← YOUR 2D BUILDING ART (visible)
│   └── billboard: true
│   └── texture: building_sprite.png
├── StaticBody3D ← 3D COLLISION BOX (invisible)
│   └── CollisionShape3D (BoxShape3D)
│       └── size: Vector3(4, 3, 4)  # Width, HEIGHT, Depth
└── NavigationLink3D ← CLIMBING PATH (invisible)
    └── connects ground to roof at Y=3
```

## 📊 WHAT NEEDS TO CHANGE

### **Must Convert to 3D (for height features):**
| Component | From | To | Reason |
|-----------|------|-----|--------|
| Units | CharacterBody2D | CharacterBody3D | Need Y position for height |
| Positions | Vector2 | Vector3 | Need Y coordinate |
| Buildings | StaticBody2D | StaticBody3D | Need vertical collision |
| Camera | Camera2D | Camera3D | View 3D space |
| Selection | 2D raycast | 3D raycast | Click on 3D objects |
| Vision rendering | 2D lines | 3D lines/meshes | Draw in 3D space |

### **Can Stay 2D (visuals):**
- ✅ All sprite art (units, buildings, props)
- ✅ UI elements
- ✅ Ground textures/tiles
- ✅ Art pipeline stays the same

### **Code Changes Required:**
- ~30% of codebase (mostly mechanical Vector2→Vector3)
- 0% of game logic (flee AI, combat, etc. transfers directly)

```gdscript
extends Camera3D  # Was Camera2D

func _ready():
    # Isometric-style view
    position = Vector3(0, 15, 10)  # Above and behind
    rotation_degrees = Vector3(-45, 0, 0)  # Look down
    fov = 70

func _process(delta):
    var move_speed = 20.0
    
    # Pan camera (WASD or arrow keys)
    if Input.is_action_pressed("camera_up"):
        position.z -= move_speed * delta  # Was position.y
    if Input.is_action_pressed("camera_down"):
        position.z += move_speed * delta
    if Input.is_action_pressed("camera_left"):
        position.x -= move_speed * delta
    if Input.is_action_pressed("camera_right"):
        position.x += move_speed * delta
    
    # Zoom (mouse wheel)
    if Input.is_action_just_released("zoom_in"):
        position.y -= 2.0  # Move camera closer
    if Input.is_action_just_released("zoom_out"):
        position.y += 2.0  # Move camera farther

func get_mouse_world_position() -> Vector3:  # Was Vector2
    var mouse_pos = get_viewport().get_mouse_position()
    var from = project_ray_origin(mouse_pos)
    var to = from + project_ray_normal(mouse_pos) * 1000
    
    # Raycast to Y=0 plane (ground)
    var plane = Plane(Vector3.UP, 0)
    return plane.intersects_ray(from, to - from)
```

**File: `building.gd`**
```gdscript
extends StaticBody3D
class_name Building

## Building properties
@export var building_sprite: Texture2D
@export var building_size: Vector3 = Vector3(4, 3, 4)  # W, H, D
@export var has_roof: bool = true
@export var roof_height: float = 3.0
@export var has_ladder: bool = true
@export var ladder_position: Vector3 = Vector3(2, 0, 0)

@onready var sprite = $Sprite3D
@onready var collision_shape = $CollisionShape3D
@onready var roof_area = $RoofArea if has_roof else null
@onready var climb_link = $NavigationLink3D if has_ladder else null

func _ready():
    # Setup visual sprite
    sprite.texture = building_sprite
    sprite.billboard = BaseMaterial3D.BILLBOARD_ENABLED
    sprite.pixel_size = 0.01
    sprite.offset = Vector2(0, -building_size.y / 2.0)  # Anchor at bottom
    
    # Setup collision (invisible 3D box)
    var box = BoxShape3D.new()
    box.size = building_size
    collision_shape.shape = box
    collision_shape.position.y = building_size.y / 2.0  # Center vertically
    
    # Setup roof area
    if roof_area:
        roof_area.position.y = roof_height
        var roof_box = BoxShape3D.new()
        roof_box.size = Vector3(building_size.x - 0.2, 0.2, building_size.z - 0.2)
        roof_area.get_node("CollisionShape3D").shape = roof_box
    
    # Setup ladder
    if climb_link:
        climb_link.start_position = ladder_position
        climb_link.end_position = ladder_position + Vector3(0, roof_height, 0)

func is_on_roof(unit: Unit) -> bool:
    if not roof_area:
        return false
    return roof_area.overlaps_body(unit)
```

**File: `human.gd`**
```gdscript
# Add height bonus to vision range
func can_see_unit(target: Unit) -> bool:
    # Calculate height advantage
    var my_height = global_position.y
    var terrain_height = get_terrain_height_at(global_position)
    var height_above_terrain = my_height - terrain_height
    
    # Height bonus: +20px vision per 1 unit of height
    var height_bonus = height_above_terrain * 20.0
    
    # Effective vision range
    var effective_range: float
    match current_state:
        State.IDLE:
            effective_range = idle_vision_radius + height_bonus
        State.SENTRY:
            effective_range = sentry_vision_range + height_bonus
        State.FLEEING:
            effective_range = flee_vision_range + height_bonus
    
    # Check if in range
    var distance = global_position.distance_to(target.global_position)
    if distance > effective_range:
        return false
    
    # Check line of sight (3D raycast handles height automatically)
    return has_line_of_sight_to(target)

# Helper: get terrain height at position
func get_terrain_height_at(pos: Vector3) -> float:
    # Raycast down to find ground
    var space_state = get_world_3d().direct_space_state
    var from = pos + Vector3(0, 10, 0)  # Start above
    var to = pos - Vector3(0, 10, 0)    # End below
    
    var query = PhysicsRayQueryParameters3D.create(from, to)
    query.collision_mask = 1  # Layer 1 (terrain/buildings)
    
    var result = space_state.intersect_ray(query)
    if result:
        return result.position.y
    return 0.0  # Default ground level
```

## 🎯 SUCCESS CRITERIA

**Phase 1 Complete When:**
- ✅ Game runs without errors in 3D mode
- ✅ All units move correctly at Y=0
- ✅ Camera pan/zoom works in 3D
- ✅ Unit selection works
- ✅ Vision arcs display correctly
- ✅ Combat/flee mechanics unchanged

**Phase 2 Complete When:**
- ✅ Buildings render as 2D sprites
- ✅ Buildings have working 3D collision
- ✅ Zombies can pathfind to roofs
- ✅ Units can stand on roofs (Y > 0)

**Phase 3 Complete When:**
- ✅ Snipers have extended vision from roofs
- ✅ Height advantage affects combat
- ✅ Ravines/hills affect gameplay
- ✅ Pathfinding works across height levels

## 🔄 ROLLBACK PLAN

**If conversion fails or takes too long:**

1. Keep 2D version in separate branch:
   ```bash
   git checkout -b 2d-version
   git commit -a -m "2D baseline before 3D conversion"
   ```

2. Can always revert:
   ```bash
   git checkout 2d-version
   ```

3. Or finish game in 2D and convert later for polish

**The conversion is NOT mandatory for a fun game!** Height features are a nice-to-have, not required for core gameplay.

## 💡 FINAL RECOMMENDATIONS

1. **Finish core 2D gameplay first** before converting
   - Get flee mechanics working perfectly
   - Implement special zombie types
   - Polish the game loop
   - Playtest extensively

2. **Convert to 3D during polish phase**
   - Once gameplay is solid
   - When you have time for 2-3 weeks of work
   - As a visual upgrade, not core requirement

3. **Start with Phase 1 only**
   - Get everything working in 3D at Y=0
   - Play it for a few days
   - Make sure it feels right
   - THEN add height features

4. **Consider hybrid approach**
   - Core game in 2D for quick iteration
   - Port to 3D for final release
   - Best of both worlds

**The 2D game can be great without height!** Don't let 3D conversion block shipping a fun game.
